Arquitetura do Sistema
O sistema usa o rádio interno do micro:bit. Cada transmissor emite continuamente a sua letra num grupo fixo. O recetor varre os grupos um a um — quando está no grupo certo e perto do transmissor, recebe a mensagem e mostra o símbolo.

Padrões LED — Referência Visual
Cada símbolo no ecrã 5×5 do micro:bit. Os participantes têm uma carta de referência impressa com estes símbolos.
Recetor 1 por grupo
Este código corre no micro:bit que cada grupo leva consigo durante a atividade. Os botões A e B percorrem os canais de 1 a 6. Quando o micro:bit está no canal de uma estação próxima, recebe a letra, faz um beep duplo e mostra o símbolo rúnico durante 3 segundos.
✅ Precisas de carregar este código em 2 ou 3 micro:bits — um por grupo de participantes.
recetor.py 
# ============================================================
#  RECETOR — Geocaching Digital: O Legado do Lidador
#  Escola Secundária da Maia · 20 março 2026
# ============================================================
#  Botão A  → desce canal (6 → 5 → 4 → 3 → 2 → 1)
#  Botão B  → sobe canal  (1 → 2 → 3 → 4 → 5 → 6)
#  Quando recebe sinal:
#    1. Beep duplo de confirmação
#    2. Mostra símbolo rúnico durante 3 segundos
#    3. Volta a mostrar o número do canal
# ============================================================

from microbit import *
import radio
import music

# ── Padrões rúnicos (5×5) ──────────────────────────────────
# 9 = LED aceso · 0 = LED apagado · ':' = separador de linha

RUNAS = {
    "L": Image("09000:09000:99900:09090:09000"),  # ᛚ Laguz
    "I": Image("09090:00900:99999:00900:09090"),  # ✳ Asterisco
    "D": Image("90009:09090:00900:09090:90009"),  # ᛞ Dagaz
    "A": Image("00900:00900:09990:90900:00900"),  # ᚨ Ansuz
    "O": Image("00900:09090:90009:09090:90009"),  # ᛟ Othala
    "R": Image("09090:99999:09090:99999:09090"),  # # Hash
}

# Canal inicial (1–6)
canal = 1
radio.config(group=canal, power=0)
radio.on()

def mostrar_canal():
    display.show(str(canal))

def subir_canal():
    global canal
    if canal < 6:
        canal += 1
        radio.config(group=canal)
        mostrar_canal()

def descer_canal():
    global canal
    if canal > 1:
        canal -= 1
        radio.config(group=canal)
        mostrar_canal()

# Mostrar canal inicial ao ligar
mostrar_canal()

# ── Loop principal ──────────────────────────────────────────
while True:

    if button_a.was_pressed():
        descer_canal()

    if button_b.was_pressed():
        subir_canal()

    msg = radio.receive()
    if msg and msg in RUNAS:
        # Beep duplo de confirmação
        music.pitch(880,  duration=150, wait=True)
        sleep(60)
        music.pitch(1320, duration=250, wait=True)

        # Mostrar símbolo rúnico 3 segundos
        display.show(RUNAS[msg])
        sleep(3000)

        # Voltar ao número do canal
        mostrar_canal()

    sleep(50)
💡 Canais 1 a 6
O ecrã mostra sempre o canal atual (1–6). Os participantes navegam com A (descer) e B (subir) até encontrarem sinal.
🔔 Beep duplo
O micro:bit emite dois tons (880 Hz + 1320 Hz) quando deteta sinal — confirmação sonora inconfundível mesmo em ambiente barulhento.
⏱️ 3 segundos de exibição
O símbolo fica visível 3s — tempo suficiente para o participante ver, consultar a carta e anotar a letra, sem ser excessivo.
🔄 Retoma o canal
Após mostrar o símbolo, o recetor retoma automaticamente a escuta no mesmo canal — o participante pode continuar sem reiniciar nada.
Transmissores 6 ficheiros
Cada transmissor é programado uma vez e depois escondido. Não precisa de interação — liga-se e fica a emitir em loop. Abaixo estão os 6 códigos, prontos a copiar para o MakeCode Python.
⚠️ Cada transmissor tem um canal diferente (1–6). Certifica-te que programas o ficheiro certo em cada micro:bit antes de os esconder!
Como Carregar o Código
O método mais simples é usar o editor online MakeCode. Não é preciso instalar nada.
🌐 Editor online: python.microbit.org — funciona em qualquer browser, incluindo Chrome e Firefox.
  1. 1 Abrir python.microbit.org no browser
  2. 2 Copiar o código do transmissor pretendido (separador Transmissores) e colar no editor
  3. 3 Ligar o micro:bit ao PC via cabo USB
  4. 4 Clicar em Connect e selecionar o micro:bit
  5. 5 Clicar em Flash — o LED amarelo no micro:bit pisca durante a transferência
  6. 6 Quando terminar, o micro:bit reinicia automaticamente e começa a emitir — verifica que o símbolo aparece no ecrã
✅ Dica: Etiqueta cada micro:bit com fita adesiva e caneta após programar — escreve o canal e a letra (ex: "C3 · D"). Evita confusões na hora de esconder.
🔋 Autonomia das pilhas
Com power=0 e emissão a cada 1s, cada transmissor dura facilmente 4–6 horas com pilhas AAA novas. Para segurança, usar pilhas novas no dia.
📡 Aumentar alcance (se necessário)
Se o espaço for muito grande, alterar power=0 para power=3 no transmissor aumenta o alcance para ~10m.
🔁 Reiniciar transmissor
Se um transmissor parecer ter parado, pressionar o botão de reset na parte de trás do micro:bit (pequeno botão junto ao USB).
💻 Alternativa: offline
Podes também usar o Mu Editor (instalado) para carregar o código via ficheiro .py sem necessitar de internet.
Testar o Sistema
Passos para verificar que tudo funciona antes de esconder os transmissores.
Protocolo de teste (1h antes da atividade)
 1. Ligar TODOS os transmissores e verificar que cada um
    mostra o símbolo correto no ecrã.

 2. Com um recetor, navegar canal a canal (B para subir):
    Canal 1 → deve receber "L" (símbolo Laguz)
    Canal 2 → deve receber "I" (símbolo Asterisco)
    Canal 3 → deve receber "D" (símbolo Dagaz)
    Canal 4 → deve receber "A" (símbolo Ansuz)
    Canal 5 → deve receber "O" (símbolo Othala)
    Canal 6 → deve receber "R" (símbolo Hash)

 3. Em cada canal, verificar que o beep ocorre
    e que o símbolo é exibido corretamente 3 segundos.

 4. Testar a distância: afastar-se do transmissor até
    deixar de receber — deve ser ~1-2m com power=0.

 5. Esconder os transmissores. Após esconder, repetir
    o teste de rececão no local exato do esconderijo.
Problemas Comuns
ProblemaCausa provávelSolução
Recetor não recebe nada no canal X Transmissor do canal X desligado ou com pilhas fracas Verificar pilhas; reiniciar com botão de reset
Símbolo errado aparece Transmissor programado com código errado Verificar CANAL e LETRA no início do ficheiro; reprogramar
Recetor recebe sinal do canal errado Dois transmissores com o mesmo grupo Verificar que cada transmissor tem CANAL único (1–6)
Alcance demasiado curto (<50cm) power=0 é demasiado restritivo no espaço Alterar para power=1 ou power=2
Alcance demasiado longo (>5m) power muito alto, transmissor em local aberto Manter power=0; colocar o transmissor dentro de uma caixa
Beep não funciona micro:bit V1 sem altifalante integrado Usar micro:bit V2, ou remover as linhas music.pitch() (funciona sem som)
